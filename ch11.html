1. Creating a Basic Table Structure
    1. Tables Defined
      1. A section of information broken up into columns and/or rows of blocks called cells.
    3. Basic Structure Tags
      1. <table></table>:Contains entire table
      2. <tr></tr>: Defines table rows
      3. <th></th>: Defines table header cells (all content is bold and centered by default)
      4. <td></td>: Defines table cells
    5. Cell Content
      1. Should be entered between opening and closing td or th tags (text and images).
      2. Can use other tags such as <strong>
      3. Browsers may wrap content according to available space in window.
      4. Use a style sheet to format groups of cells.
  3. Formatting Tables
    1. Borders and Margins
      1. All tables have internal and external borders by default.
      2. Use border attribute in the table tag to turn all internal and external borders on or off.
      3. Use border-collapse property in style sheetl to adjust general table spacing.
      4. Use padding and margin properties in style sheet to format spacing in and around cells (but individual cells can only have padding in CSS, no margins).
    3. Width and Height
      1. Can be specified in absolute (pixel measurements) or relative sizes (percentages).
      2. Must weigh advantages against disadvantages of each when determining which method to use.
      3. Preferred method is to use the width and height properties, attached to the table tag selector (to size an entire table) in your style sheet.
      4. Can also use the height and width attributes in your table HTML to size individual cells.
    5. Basic Alignment
      1. Add the float property to your table's style declaration to align the entire table to one side or the other (left or right).
      2. Remember: The floated table must be placed before the rest of the content in the HTML code.
      3. More complex placement of a table on the page can be achieved with CSS positioning, as discussed in Chapter 6.
    7. Color
      1. Add the background-color property to the table tag declaration to specify a background color for the entire table.
      2. Can also add the background-color property to individual cells.
      3. Use CSS classes to apply a background color to groups of cells.
    9. Background Images
      1. Add the background-image property to the table tag style declaration to specify a background image for the entire table.
      2. Can also add the background-image property to individual cells.
      3. Use CSS classes to apply a background image to groups of cells.
    11. Captions
      1. Add the caption tag after the table tag to specify a caption for the entire table.
        1. Add the text-align property in your style sheet to adjust how the text is aligned within the caption.
        2. Add the caption-side property in your style sheet to customize where the caption appears in relation to the table.
        3. Caption text is placed between the opening and closing caption tags.
      3. Each browser handles this a little differently--some show the caption below the table by default, some show it above the table by default, while others don't show it at all.
  5. Formatting Content Within Table Cells
    1. Attributes/Properties
      1. Use most of the same attributes and properties to format cells as you do to format whole tables
  7. Alignment
    1. Adding the text-align or vertical-align property to the tr tag style declaration affects all cells in that row.
    2. Adding the text-align or vertical align property to the td or th tag style declarations affects the contents of that cell.
    3. text-align: Default value is left (except for th cells, where it's center).
    4. vertical-align: Default value is middle.
    5. Refer to Table 11-1 in Chapter 11 for additional values.
    6. Use classes to align groups of cells.
  9. Width and Height
    1. Adding the width or height properties to the tr tag style declaration affects all cells in that row.
    2. Adding the width or height properties to the td or th tag style declarations affects the contents of that cell.
  11. Cell Padding
    1. Use the padding property to adjust the space around the content in each cell.
    2. Padding can be adjusted as a whole (with one dimension) or individually (with two to four dimensions).
  13. Colors
    1. Adding the background-color property to the tr tag style declaration affects all cells in that row.
    2. Adding the background-color property to the td or th tag style declarations affects the contents of that cell.
    3. Each browser handles this a little differently--some show the background color in the table borders and others don/t.
    4. Where the color begins and ends is affected by the table's cell padding and cell spacing.
    5. Use classes to color groups of cells.
  15. Prohibiting Line Breaks
    1. Add white-space property (with a value of nowrap) to the th or td tag style declaration to try and force the browser not to wrap the cell/s contents
    2. This is not foolproof; there are times when the browser may decide to do it anyway (depending on the width of the browser window).
  17. Spanning Columns
    1. Add the colspan attribute to the th or td tag to force a cell to cover multiple columns.
    2. Don't include another set of td tags for the "missing" cells.
  19. Spanning Rows
    1. Add the rowspan attribute to the th or td tag to force a cell to cover multiple rows.
    2. Don't include another set of td tags for the "missing" cells.
  21. Using Additional Formatting Techniques
    1. Grouping Rows
      1. Use the thead tags to surround the header information of a table.
      2. Use the tfoot tags to surround the footer information of a table.
      3. Use the tbody tags to surround the body information of a table
      4. These tags make tables more accessible.
      5. If you include a thead and/or a tfoot, you must include at least one tbody.
      6. Use these tags as selectors in your style sheet to style whole sections of a table at once.
    3. Grouping Columns
      1. Use the colgroup tag to group certain columns in a table.
        1. span: How many columns the group spans
        2. width: The width of all columns in the group
        3. align: The horizontal alignment of all columns in the group
        4. valign: The vertical alignment of all columns in the group
      3. The colgroup tag doesn't contain any tr or td tags, only col tags.
      4. Use the col tag to identify individual column details as needed to override the general settings specified in the colgroup tag.
